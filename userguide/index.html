The zeroculture.co.uk theme by de.lin.e.ate has been designed and the protoype built to take advantage of Metafizzy's Packery script to display zeroculture articles as items (article-list__item) of varied width and heights in a grid layout.

The article-list (without Packery) appears in its default state above the footer on the single article template as one row and 4 columns with items of equal width and height

The .article-list with Packery

To lay out the article-list with Packery, add the packery and js-packery classes to the article-list 

<div class="article-list  packery  js-packery">

  <div class="article-list__item">...</div>
  <div class="article-list__item">...</div>
  <div class="article-list__item">...</div>
  <div class="article-list__item">...</div>

</div> <!-- /.article-list -->

The article-list with Packery appears on the front page template and the archive template as multiple rows and columns with items of varied width and height

The .article-list__item

The article-list__item is a child element of article-list

It contains the individual article content, including article image and title

  </article>
</div> <!-- /.article-list__item -->

How to change article item width and height

The Packery article list has items of varied width and height.

Change the width and height of article list items with classes

Width

add width class to <div class="article-list__item">

  • <div class="article-list__item"> = 25% default
  • <div class="article-list__item w1-2"> = 50%
  • <div class="article-list__item w3-4"> = 75%
  • <div class="article-list__item w1-1"> = 100%

Height

change <article>'s default img-bkg-sm height class

  • <article class="article img-bkg-sm"> = 240px small/default
  • <article class="article img-bkg-md"> = 480px medium
  • <article class="article img-bkg-lg"> = 720px large

The CSS

project-zero/css/main.css view on github

/*------------------------------------------------*
  # ARTCILE LIST ITEM
\*------------------------------------------------*/

.article-list__item {
  position: relative;
  width: 50%;
  float: left;
  background: #CCC;
}

.article-list__item.w1-1 { width:  100%; }

.img-bkg-sm,
.img-bkg-md,
.img-bkg-lg {
  position: relative;
  background-repeat: no-repeat;
  background-position: center center;
  background-size: cover;
}

.img-bkg-sm {
  height: 240px;
}

@media screen and (min-width: 600px) {
  .article-list__item { width: 25%; }
  .article-list__item.w1-2 { width:  50%; }
  .article-list__item.w3-4 { width:  75%; }
  .article-list__item.w1-1 { width:  100%; }
   }

  .img-bkg-md { height: 480px; }
  .img-bkg-lg { height: 720px; }
}

Though an article-list will look fine without Packery, width and heights of article items may need micro-managing to control layout that relies only on CSS to control the white space.

If an article-list layout is controlled by Packery, Packery will override .article-list__item CSS styles with its own. For example, it will add position: absolute; as an inline style inside the markup to position each .article-list__item child element.
